📝 Project: MBTI 기반 소통 시뮬레이션 웹앱 (MVP Phase 1)
1. 프로젝트 개요 (Overview)
본 프로젝트는 사용자가 타 MBTI 성향을 가진 AI 페르소나와 모의 대화를 나누며 소통 방식을 이해하고 개선할 수 있도록 돕는 웹 애플리케이션입니다.

최종 목표: 다중 사용자 실시간 채팅 번역 및 수익화 모델(B2B, 토큰) 구축

현재 목표 (MVP - Phase 1): 싱글 플레이어 기반의 핵심 AI 대화 및 첨삭 기능(대화 실전 모드, 스킬 숙련 모드)의 빠른 구현과 사용성 검증

2. 전체 시스템 아키텍처 (High-Level Architecture)
향후 확장성(멀티플레이, 결제 등)을 고려하여 프론트엔드와 백엔드를 분리하는 구조를 채택하되, MVP 단계에서는 복잡한 DB 설계보다 LLM API 연동에 집중합니다.

Frontend (Client): * 사용자 인터페이스 (MBTI 선택, 채팅창 UI) 제공

상태 관리 (대화 기록 임시 저장)

Backend (API Server):

클라이언트 요청 수신 및 프롬프트 조합

LLM API (OpenAI 등) 통신 및 응답 파싱

Database (향후 확장용): MVP 단계에서는 유저 인증 없이 LocalStorage 또는 세션 기반 임시 저장을 활용하고, 추후 RDBMS(PostgreSQL 등) 도입.

3. 기술 스택 제안 (AI 코드 생성에 최적화)
Frontend: Next.js (React), Tailwind CSS (빠른 UI 구현)

Backend: Next.js API Routes (프론트/백 통합 보일러플레이트로 초기 세팅 시간 단축)

AI API: OpenAI API (gpt-3.5-turbo 또는 gpt-4o-mini - 빠르고 저렴한 MVP용)

4. 사용자 스토리 및 기능 요구사항 (Phase 1 Scope)
Epic 1: 대화 실전 모드 (Practical Chat)
"사용자는 특정 MBTI를 가진 가상의 상대와 대화하며 반응을 확인하고 싶다."

기능 1.1: 페르소나 설정

사용자는 16가지 MBTI 중 하나를 선택할 수 있다.

선택된 MBTI에 가상의 이름(예: "깐깐한 기획자 INTJ")을 부여할 수 있다.

기능 1.2: 1:1 채팅 UI

사용자가 메시지를 입력하고 전송할 수 있다.

AI가 해당 MBTI의 성향, 말투, 사고방식을 반영하여 응답한다.

기능 1.3: 대화 컨텍스트 유지

이전 대화 내역(최대 N개)을 기억하여 문맥에 맞는 자연스러운 대화를 이어간다.

Epic 2: 스킬 숙련 모드 (Skill Training)
"사용자는 내가 하고 싶은 말을 상대 MBTI가 듣기 좋게 교정받고, 그 이유를 학습하고 싶다."

기능 2.1: 타겟 MBTI 선택 및 입력

훈련 대상의 MBTI를 선택하고, 본인이 전하고 싶은 원문 메시지를 입력한다.

기능 2.2: AI 교정 및 피드백 출력

시스템은 원문을 대상 MBTI 성향에 맞게 교정(첨삭)한 **[결과 문장]**을 제공한다.

문장이 그렇게 수정된 **[심리적/성향적 이유]**를 하단에 별도 텍스트로 설명한다.

5. 코덱스/AI 툴을 위한 핵심 프롬프트 설계 지침 (System Prompts)
AI 개발 툴이 API 연동 코드를 작성할 때, 아래의 시스템 프롬프트 구조를 백엔드 로직에 반영하도록 지시해야 합니다.

[대화 실전 모드 용 프롬프트 구조]
"너는 지금부터 [선택된 MBTI] 성향을 가진 사람이야. 이름은 [사용자 지정 이름]이야. 사용자의 말에 대답할 때 반드시 [선택된 MBTI]의 특징적인 말투, 논리 전개 방식, 공감 능력을 반영해서 대답해."

[스킬 숙련 모드 용 프롬프트 구조]
"너는 [선택된 MBTI] 전문 소통 코치야. 사용자가 입력한 문장을 [선택된 MBTI]가 가장 긍정적으로 받아들일 수 있는 화법으로 교정해 줘. 응답은 반드시 JSON 형식으로 { "revised_sentence": "교정된 문장", "reason": "이 화법이 해당 MBTI에게 효과적인 이유 설명" } 형태로 반환해."
